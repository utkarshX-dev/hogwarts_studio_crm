# Analytics & Dashboard — Summary

This file summarizes the frontend structure, architecture, API calls, and data shapes used by the **Analytics** and **Dashboard** pages.

## Key pages
- Analytics page: app/(dashboard)/analytics/page.tsx
- Dashboard page: app/(dashboard)/dashboard/page.tsx

## Frontend architecture
- Framework: Next.js (App Router) + React + TypeScript.
- Charts: Recharts (AreaChart, BarChart, PieChart, ResponsiveContainer).
- UI primitives: `components/ui/*`; shared UI: `components/shared/*`.
- Data fetching: client-side fetch + short-interval polling (Dashboard polls every 30s).

## Main shared pieces
- `hooks/use-realtime-data.ts` — fetches `/api/realtime-data` and exposes `{ invoices, analytics, loading, error, refresh }`.
- `components/shared/StatCard.tsx` — statistic card used across both pages.
- `components/shared/PageHeader.tsx` — page title and actions container.
- `components/shared/ShimmerLoader.tsx` — skeleton loaders used while data loads.
- `components/shared/Badges.tsx` — badges for lead/payment/status visuals.

## Backend / API routes used
- `GET /api/realtime-data` — aggregation endpoint that returns `{ success, invoices, analytics }`.
  - See: app/api/realtime-data/route.ts
  - Aggregates: payments, clients (leads), editing, shoots.
  - Produces: `analytics` with `totalRevenue`, `totalProjects`, `avgProject`, `conversionRate`, `REVENUE_DATA`, `SERVICE_DISTRIBUTION`, `STATUS_DISTRIBUTION`, `SALES_TREND`, `EDITORS`.

- `GET /api/shoots` — returns shoots (role-filtered). See: app/api/shoots/route.ts
- `GET /api/editing` — returns editing projects with latest revision feedback. See: app/api/editing/route.ts
- `GET|POST|PUT /api/clients` — read/create/update leads with role checks. See: app/api/clients/route.ts

## Sheets integration
- `lib/google/sheets.ts` reads Google Sheets and converts rows to typed objects (Lead, Payment, Shoot, EditingProject).
- Caching: in-memory cache + request coalescing (TTL ≈ 5s) to reduce sheet reads.

## Data shapes (summary)
- analytics: {
  - totalRevenue: number
  - totalProjects: number
  - avgProject: number
  - conversionRate: number
  - REVENUE_DATA: [{ month: string, revenue: number, projects: number }]
  - SERVICE_DISTRIBUTION: [{ name: string, value: number, color: string }]
  - STATUS_DISTRIBUTION: [{ name: string, value: number, color: string }]
  - SALES_TREND: [{ month: string, leads: number, converted: number }]
  - EDITORS: [{ id, name, initials, email, status, activeProjects, completedProjects, specialization[] }]
  }

- invoices: [{ id, projectId, clientName, amount, status, dueDate, paidDate, type }]

## What each page renders (short)
- Analytics:
  - Top stat cards (Revenue, Projects, Avg Project, Conversion Rate)
  - Monthly Revenue (AreaChart)
  - Service Distribution (PieChart)
  - Sales Funnel (BarChart)
  - Project Status Distribution (PieChart)
  - Editor Performance list

- Dashboard:
  - Top stat grid (6 StatCards: Revenue, Active Projects, Today's Shoots, Pending Reviews, Pending Payments, Editors Busy)
  - Revenue Trend (AreaChart) — link to Analytics
  - Project Pipeline counts
  - Recent Projects list (up to 5)
  - Upcoming Shoots grid

## Behavior & important notes
- Role-based filtering: API routes call `getAuthenticatedUser()` and filter rows for roles (`sales`, `editor`, `shoot`, etc.).
- Error handling: `/api/realtime-data` returns `{ success: false, error }` on failure; the analytics UI shows a retry button.
- Formatting: money & dates use `lib/formatter` helpers.
